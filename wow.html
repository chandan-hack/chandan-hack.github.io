Database Management System

Introduction

A Database Management System (DBMS) is a software system that allows users to create, define, manipulate and manage databases. It provides a way for organizations to store, organize and retrieve large amounts of data quickly and efficiently in an organized manner. A database management system (DBMS) is a software system that handles the storage, retrieval, and manipulation of data. It acts as an interface between users and databases, allowing for efficient data operations.

  • Data definition: Creating and modifying database structures
  • Data manipulation: Inserting, updating, and deleting data
  • Data querying: Retrieving specific information
  • Access control: Managing user permissions and security

What is data in DBMS?

In a Database Management System (DBMS), data refers to the actual information stored within the database, which can be organized into various formats like text, numbers, and dates. It's the core component that the DBMS manages and provides access to. Essentially, data is the collection of facts, figures, or details that the database is designed to hold and manage.

Software Development

i) Database Design

Database design is the foundational process of structuring an organization's data to support its information needs effectively and efficiently. It's an iterative process that begins long before a single line of code is written and directly impacts the performance, scalability, and maintainability of any software system.

ii) Database Administration

Database Administration encompasses the comprehensive management, maintenance, and operational support of database systems throughout their lifecycle. While database design focuses on the initial structure, DBA is about keeping the database healthy, secure, and performing optimally in a production environment.

Adding new records to the database. R - Read: Corresponds to the SELECT DQL command. Retrieving data from the database. U - Update: Corresponds to the UPDATE DML command. Modifying existing records. Keys in Database: Keys are crucial for establishing relationships between tables and ensuring data integrity and uniqueness within a relational database.

  • Primary Key
    • Uniquely identifies each record in a table.
    • Must contain unique values for each row.
    • Cannot contain NULL values (NOT NULL constraint).
    • A table can have only one primary key.
    • Often implemented with an auto-incrementing integer or a UUID (Universally Unique Identifier).

Anomalies in Database

Database anomalies are flaws in the design of a relational database that can lead to data redundancy and inconsistencies. They typically arise in poorly normalized databases.

  • Insertion Anomaly: Occurs when you cannot insert a new record into a table without adding information about another unrelated entity.
  • Deletion Anomaly: Occurs when deleting a record results in the loss of other, unrelated, important information.
  • Update Anomaly: Occurs when modifying an attribute value requires updating multiple rows, leading to potential inconsistencies if all copies are not updated.
